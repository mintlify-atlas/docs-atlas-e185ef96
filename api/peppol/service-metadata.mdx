---
title: 'Service metadata'
description: 'Peppol SMP 1.x endpoints for reading, creating, and deleting service metadata (access point endpoints per document type).'
---

Service metadata describes which access point endpoints a participant supports for a given document type and process. It is keyed by the combination of participant identifier and document type identifier.

XML namespace: `http://busdox.org/serviceMetadata/publishing/1.0/`

---

## GET `/{ServiceGroupId}/services/{DocumentTypeId}`

Returns the signed service metadata for the given participant and document type. The response is signed with the SMP's private key using XML-DSig.

**Authentication:** Not required.

<ParamField path="ServiceGroupId" type="string" required>
  The participant identifier in `scheme::value` format, URL-encoded.
</ParamField>

<ParamField path="DocumentTypeId" type="string" required>
  The document type identifier in `scheme::value` format, URL-encoded. Example: `busdox-docid-qns%3A%3Aurn%3Aoasis%3Anames%3Aspecification%3Aubl%3Aschema%3Axsd%3AInvoice-2%3A%3AInvoice%23%23urn%3Acen.eu%3Aen16931%3A2017%23compliant%23urn%3Afdc%3Apeppol.eu%3A2017%3Apoacc%3Abilling%3A3.0%3A%3A2.1`
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl https://smp.example.org/iso6523-actorid-upis%3A%3A0088%3A5060210000007/services/busdox-docid-qns%3A%3Aurn%3Aoasis%3Anames%3Aspecification%3Aubl%3Aschema%3Axsd%3AInvoice-2%3A%3AInvoice%23%23urn%3Acen.eu%3Aen16931%3A2017%23compliant%23urn%3Afdc%3Apeppol.eu%3A2017%3Apoacc%3Abilling%3A3.0%3A%3A2.1
    ```
  </Tab>
  <Tab title="Response">
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>

          <ParticipantIdentifier scheme="iso6523-actorid-upis">0088:5060210000007</ParticipantIdentifier>
    ```

### Response fields

<ResponseField name="SignedServiceMetadata" type="object">
  <Expandable title="SignedServiceMetadata">
    <ResponseField name="ServiceMetadata" type="object">
      Contains either a `ServiceInformation` element (for direct endpoints) or a `Redirect` element (pointer to another SMP).
    </ResponseField>
    <ResponseField name="Signature" type="object">
      XML-DSig signature over the `ServiceMetadata` element, generated by the SMP server using its configured keystore.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="ServiceInformation" type="object">
  <Expandable title="ServiceInformation">
    <ResponseField name="ParticipantIdentifier" type="string">The participant identifier.</ResponseField>
    <ResponseField name="DocumentIdentifier" type="string">The document type identifier.</ResponseField>
    <ResponseField name="ProcessList" type="array">
      List of supported processes, each containing a `ServiceEndpointList`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="Endpoint" type="object">
  <Expandable title="Endpoint attributes and child elements">
    <ResponseField name="transportProfile" type="string">Transport profile ID, e.g. `peppol-transport-as4-v2_0`.</ResponseField>
    <ResponseField name="EndpointReference/Address" type="string">URL of the access point.</ResponseField>
    <ResponseField name="RequireBusinessLevelSignature" type="boolean">Whether a business-level signature is required.</ResponseField>
    <ResponseField name="ServiceActivationDate" type="date">Date from which the endpoint is active (ISO 8601).</ResponseField>
    <ResponseField name="ServiceExpirationDate" type="date">Date after which the endpoint is inactive (ISO 8601).</ResponseField>
    <ResponseField name="Certificate" type="string">Base64-encoded DER X.509 certificate of the access point.</ResponseField>
    <ResponseField name="ServiceDescription" type="string">Human-readable description.</ResponseField>
    <ResponseField name="TechnicalContactUrl" type="string">Contact URL or mailto for technical issues.</ResponseField>
    <ResponseField name="TechnicalInformationUrl" type="string">Optional URL for further technical information.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
|--------|-----------|
| `400` | Identifier cannot be parsed |
| `404` | Service group or service metadata not found, or no endpoints are registered |

## PUT `/{ServiceGroupId}/services/{DocumentTypeId}`

Creates or replaces the service metadata for the given participant and document type. Provide either a `ServiceInformation` block (direct endpoints) or a `Redirect` block.

**Authentication:** Required. See [Authentication](/api/authentication).

<ParamField path="ServiceGroupId" type="string" required>
  The participant identifier in `scheme::value` format, URL-encoded.
</ParamField>

<ParamField path="DocumentTypeId" type="string" required>
  The document type identifier in `scheme::value` format, URL-encoded.
</ParamField>

<Tabs>
  <Tab title="Service information body">
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <ServiceMetadata xmlns="http://busdox.org/serviceMetadata/publishing/1.0/"
                     xmlns:id="http://busdox.org/transport/identifiers/1.0/">
      <ServiceInformation>
        <ParticipantIdentifier scheme="iso6523-actorid-upis">0088:5060210000007</ParticipantIdentifier>
        <DocumentIdentifier scheme="busdox-docid-qns">urn:oasis:names:specification:ubl:schema:xsd:Invoice-2::Invoice##urn:cen.eu:en16931:2017#compliant#urn:fdc:peppol.eu:2017:poacc:billing:3.0::2.1</DocumentIdentifier>
        <ProcessList>
          <Process>
            <ProcessIdentifier scheme="cenbii-procid-ubl">urn:fdc:peppol.eu:2017:poacc:billing:01:1.0</ProcessIdentifier>
            <ServiceEndpointList>
              <Endpoint transportProfile="peppol-transport-as4-v2_0">
                <EndpointReference xmlns="http://www.w3.org/2005/08/addressing">
                  <Address>https://ap.example.org/as4</Address>
                </EndpointReference>
                <RequireBusinessLevelSignature>false</RequireBusinessLevelSignature>
                <ServiceActivationDate>2024-01-01</ServiceActivationDate>
                <ServiceExpirationDate>2026-12-31</ServiceExpirationDate>
                <Certificate>MIIBkTCB+wIJ...</Certificate>
                <ServiceDescription>Peppol BIS Billing 3.0</ServiceDescription>
                <TechnicalContactUrl>mailto:ap-admin@example.org</TechnicalContactUrl>
              </Endpoint>
            </ServiceEndpointList>
          </Process>
        </ProcessList>
      </ServiceInformation>
    </ServiceMetadata>
    ```
  </Tab>
  <Tab title="Redirect body">
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <ServiceMetadata xmlns="http://busdox.org/serviceMetadata/publishing/1.0/"
                     xmlns:id="http://busdox.org/transport/identifiers/1.0/">
      <Redirect
        href="https://other-smp.example.org/iso6523-actorid-upis%3A%3A0088%3A5060210000007/services/..."
        certificateUID="some-cert-uid"/>
    </ServiceMetadata>
    ```
  </Tab>
  <Tab title="curl example">
    ```bash
    curl -X PUT \
      'https://smp.example.org/iso6523-actorid-upis%3A%3A0088%3A5060210000007/services/busdox-docid-qns%3A%3Aurn%3Aoasis%3Anames%3Aspecification%3Aubl%3Aschema%3Axsd%3AInvoice-2%3A%3AInvoice%23%23urn%3Acen.eu%3Aen16931%3A2017%23compliant%23urn%3Afdc%3Apeppol.eu%3A2017%3Apoacc%3Abilling%3A3.0%3A%3A2.1' \
      -u admin@helger.com:password \
      -H "Content-Type: application/xml" \
      --data-binary @service-metadata.xml
    ```
  </Tab>
</Tabs>

<Warning>
The `ParticipantIdentifier` and `DocumentIdentifier` values inside the request body must match the identifiers in the URL. The server returns `400 Bad Request` if they differ.
</Warning>

<Note>
Endpoints with overlapping validity periods (same transport profile, overlapping `ServiceActivationDate`/`ServiceExpirationDate`) within the same process are rejected with `400 Bad Request`.
</Note>

### Response

Returns `HTTP 200 OK` with an empty body on success.

### Error responses

| Status | Condition |
|--------|-----------|
| `400` | Identifier mismatch, missing fields, or overlapping endpoint validity periods |
| `401` | Missing or invalid credentials |
| `403` | Authenticated user does not own the service group |
| `404` | Service group does not exist |

## DELETE `/{ServiceGroupId}/services/{DocumentTypeId}`

Deletes the service metadata (or redirect) for the given participant and document type.

**Authentication:** Required. See [Authentication](/api/authentication).

<ParamField path="ServiceGroupId" type="string" required>
  The participant identifier in `scheme::value` format, URL-encoded.
</ParamField>

<ParamField path="DocumentTypeId" type="string" required>
  The document type identifier in `scheme::value` format, URL-encoded.
</ParamField>

<Tabs>
  <Tab title="curl example">
    ```bash
    curl -X DELETE \
      'https://smp.example.org/iso6523-actorid-upis%3A%3A0088%3A5060210000007/services/busdox-docid-qns%3A%3Aurn%3Aoasis%3Anames%3Aspecification%3Aubl%3Aschema%3Axsd%3AInvoice-2%3A%3AInvoice%23%23urn%3Acen.eu%3Aen16931%3A2017%23compliant%23urn%3Afdc%3Apeppol.eu%3A2017%3Apoacc%3Abilling%3A3.0%3A%3A2.1' \
      -u admin@helger.com:password
    ```
  </Tab>
</Tabs>

### Response

Returns `HTTP 200 OK` on success.

### Error responses

| Status | Condition |
|--------|-----------|
| `400` | Identifier cannot be parsed |
| `401` | Missing or invalid credentials |
| `403` | Authenticated user does not own the service group |
| `404` | Service group or service metadata not found |
